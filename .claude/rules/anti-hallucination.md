# Anti-Hallucination

## Before stating API signatures, library methods, or version-specific behavior

1. Verify with the `find-docs` skill (`ctx7` CLI) or WebSearch — even for well-known libraries; training data on API details is frequently outdated
2. State the version explicitly
3. Cite the source

## Never

- Invent function signatures, parameter orders, or return types
- Guess library versions or assume API behavior without docs
- Fabricate citations, URLs, paper titles, or statistics
- Claim certainty without verification

## When uncertain

Say so. "I believe X but haven't verified" beats a confident wrong answer.
Use WebSearch for recent facts (<1 year), Read for file content, Context7 for library APIs.
